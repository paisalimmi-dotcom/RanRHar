## 2. Flow การทดลอง

### 2.1 ลูกค้าสั่งอาหาร (ไม่ต้อง Login)

1. เปิด http://localhost:3000 → redirect ไปเมนูโต๊ะ A12
2. คลิก "เพิ่ม" ในรายการอาหาร
3. ที่ตะกร้า (footer) คลิก "Place Order"
4. ตรวจสอบยอด → กด "Place Order"
5. ดูหน้า Order Success

### 2.2 พนักงานเสิร์ฟ / พ่อครัว

1. ไปที่ http://localhost:3000/staff
2. Login ด้วย staff@test.com หรือ chef@test.com
3. **KDS (ครัว):** ดูออเดอร์รอทำ → กด "รับออเดอร์" → กด "เสร็จแล้ว"
4. **ออเดอร์:** ดูรายการ อัปเดตสถานะ บันทึกการชำระ
5. **สรุปโต๊ะ:** ดูออเดอร์ตามสถานะ

### 2.3 ผู้จัดการ

1. Login ด้วย manager@test.com
2. **Admin** → จัดการเมนู: เพิ่ม/แก้ไขหมวดหมู่และรายการอาหาร
3. **สต็อก:** จัดการสินค้าคงคลัง
4. **ออเดอร์:** บันทึกการชำระได้

### 2.4 เจ้าของร้าน

1. Login ด้วย owner@test.com
2. ดู Dashboard ออเดอร์ สรุปโต๊ะ — **ดูอย่างเดียว ไม่มีสิทธิ์แก้ไข**

## 3. สิ่งที่ควรทดสอบ

| การทดสอบ | วิธี |
|----------|------|
| ลูกค้าสั่งอาหารได้ | เมนู → ตระกร้า → Checkout → Success |
| ออเดอร์แสดงใน KDS | สั่งอาหาร → ไป KDS ดูคอลัมน์ "รอทำ" |
| อัปเดตสถานะออเดอร์ | KDS กด "รับออเดอร์" หรือ "เสร็จแล้ว" |
| บันทึกการชำระ | ออเดอร์ → Record → เลือก CASH/QR |
| จัดการเมนู | Admin → จัดการเมนู (manager เท่านั้น) |
| จัดการสต็อก | สต็อก (manager, staff) |
